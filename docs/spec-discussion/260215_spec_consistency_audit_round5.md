- `docs/spec/usecases.md`

## 監査サマリ

- 重大な不整合: 0件
- 中程度の不整合: 1件（修正済み）
- 軽微な不整合: 1件（修正済み）
- PO確認が必要な論点: 0件

## 指摘一覧

### 1. Event 仕様が「前年度（closed）」表現になっており、closed の適用範囲が前年度限定に読める（中・修正済み）

**根拠**:

- `docs/spec/functions.md` の Event 仕様に「前年度（closed）に属するトランザクションは編集不可」と記載
- 一方で他セクションは「closed 年度」を一般概念として扱っている

**問題**:

- 「前年度のみロック対象」と誤解される余地があり、年度ロックの適用範囲を実装者が狭く解釈するリスクがある。

**対応内容（2026-02-15）**:

- `docs/spec/functions.md` の該当文言を「closed 年度に属するトランザクションは編集不可」に修正。

### 2. UC-5.6（イベントの編集）の制約文が、イベント編集とトランザクション編集の境界を曖昧にしている（軽微・修正済み）

**根拠**:

- `docs/spec/usecases.md` UC-5.6 の制約が「closed 年度に属するものは編集不可」のみで、対象がイベント自体かイベント紐付けかを明確にしていない。
- UC-5.2 では closed 年度トランザクションの紐付け変更不可が明記されている。

**問題**:

- イベント名称・期間の編集まで禁止と誤読されると、Event が年度非依存という定義と運用上の齟齬が生じる。

**対応内容（2026-02-15）**:

- `docs/spec/usecases.md` UC-5.6 の制約を以下へ分解して明記。
- closed 年度トランザクションのイベント紐付け変更は不可
- イベント自体（名称・期間）は年度状態に関わらず編集可能

## 補足

- 本ラウンドでは、ドメイン方針の追加判断を要する論点は検出されなかった。
- 監査時点: 2026-02-15
