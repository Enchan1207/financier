# ダッシュボード再設計

**日付**: 2026-03-14
**経緯**: 既存の `routes/index.tsx` が肥大化しており、そもそもホーム画面に何を表示すべきかを要件に立ち返って再検討した。

---

## 背景

既存のホーム画面は各ページの縮小版を詰め込んだ構成になっており、重さの原因となっていた。
また UC-7.1「ダッシュボードの表示」の定義が「財務状況の全体像を把握する」と曖昧で、
仕様策定時点でダッシュボードの役割が十分に定義されていなかった。

---

## ホーム画面の「問い」の再定義

ユーザーがホームを開いたときに持っている問いは以下の3つに整理する。

| #   | 問い                           | 頻度       |
| --- | ------------------------------ | ---------- |
| 1   | 今の財政体力はどうか？         | 週次〜月次 |
| 2   | 今月このまま行っても大丈夫か？ | 週次       |
| 3   | 支出を記録したい               | 日次       |

---

## 決定事項

### 1. 残高の表示方法

**決定**: 「実質自由資金」を主指標として大きく表示し、積立残高はサブ情報として添える。

```
実質自由資金  ¥780,800
             （＋積立残高 ¥463,000）
```

**理由**:

- 内部残高（¥1,243,800）をそのまま表示すると積立に充当済みの金額も含まれるため、
  「自由に使えるお金」という直感的な判断軸とずれる。
- 実質自由資金 = 内部残高 − 積立残高合計 を主表示にすることで、
  意思決定に直接使える数値が一目でわかる。
- 積立残高合計は「将来の支出への意思表示」として並記し、財政体力の全体像も確認できる。

---

### 2. 予定支出の扱い（近日の予定支出カードの廃止）

**決定**: 独立した「近日の予定支出」カードは廃止し、「今月の状況」カード内に
今月の予定支出を折り込んで「見込み」として表示する（案B）。

```
今月の状況

支出  ¥48,730 ──────── 65%  ── 予算 ¥75,000
           └ 今月の予定あと ¥23,000（見込み 88%）

⚠ 日用品 超過
```

**理由**:

- 未来日トランザクションは内部残高にも予算実績にも含まれない仕様（UC-1.2）のため、
  実績だけを表示すると楽観的な誤解を招くリスクがある。
- 「実績 vs 予算」ではなく「見込み vs 予算」まで示すことで、直近の予定支出を加味した
  今月の余裕が即断できる。これは本システムの「意思決定支援」という設計思想と整合する。
- 独立カードにすると情報が分断されるが、今月の状況カードに統合することで
  「今月は安全か」という問いへの答えが1か所に集約される。

**超過カテゴリの表示**:

- 警告レベル（超過・注意）のカテゴリのみ表示し、正常なカテゴリは非表示にする。
- カード内に「↓詳細」で展開できるインターフェースを設ける（全カテゴリの詳細は `/budget` へ）。

---

### 3. クイック支出登録のダイアログ化

**決定**: 常時表示のフォームを廃止し、ダイアログ（モーダル）に変更する。

- **スマホ**: 画面右下にフローティングアクションボタン（FAB）「＋」を配置
- **デスクトップ**: ヘッダーにボタンを配置

**理由**:

- 支出の記録は日次で発生する最重要アクションだが、常にフォームが表示されている必要はない。
- ホーム画面のコンテンツ密度を下げ、情報の視認性を向上させる。
- UC-1.1 のフローはダイアログ内で完結できる（種別・金額・日付・カテゴリ・名前）。

---

## UC-7.1 の再定義

### 変更前

> **概要**: 財務状況の全体像を把握する。
>
> **表示項目**:
>
> - 内部残高（全期間）
> - 積立残高合計
> - 当年度の収支サマリ

### 変更後

> **概要**: 日々の運用状態を一画面で確認し、今月の安全性を即断できる。
>
> **表示項目**:
>
> - 実質自由資金（= 内部残高 − 積立残高合計）および積立残高合計
> - 今月の支出合計 / 月次予算合計（進捗率）
> - 今月の予定支出合計および見込み進捗率
> - 予算警告カテゴリ（超過・注意のみ。正常時は非表示）
> - 最近の取引（直近 5 件）
>
> **アクション**:
>
> - 支出の記録（ダイアログ経由。スマホは FAB、デスクトップはヘッダーボタン）
