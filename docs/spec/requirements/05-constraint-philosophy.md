# 5. 制約思想

- 索引: [要求仕様書](../requirements.md)
- 関連機能: [機能仕様書](../functions.md)
- 関連ユースケース: [ユースケース定義書](../usecases.md)

---

本システムは、財務状態の良否に対して利用者の行動を制限しない。

- 予算超過を禁止 **しない**
- 積立不足を禁止 **しない**

その代わりに以下を行う:

- 常時可視化
- 警告表示
- 差分明示

ただし、以下のドメイン不変条件に違反する入力は拒否する:

- 参照整合性（存在しない/利用不可カテゴリへの登録など）
- 会計境界（closed 年度への更新など）
- 型整合性（`Transaction.type` と `Category.type` の不一致など）
