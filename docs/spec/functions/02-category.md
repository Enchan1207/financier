## 3.1 定義

トランザクションを分類するためのラベル。年度に依存せずシステム全体で共通。

## 3.3 仕様

- 階層構造を持たない（フラット）
- カテゴリ削除は物理削除とし、トランザクション・予算・積立定義の参照がすべて 0 件の場合のみ許可する
- 参照が存在する場合は削除を拒否し、参照先エンティティの種別をエラーとして通知する
- カテゴリ種別（income / expense / saving）は作成時に決定し、変更不可
- 支出カテゴリ作成時には「積立として作成する」を選択できる（`type=saving`）
- 積立として作成するかどうかはカテゴリ作成時にのみ決定でき、作成後は変更不可

### アイコン・色

- `icon` と `color` はカテゴリの視覚的識別性を高めるための属性であり、UI 表示にのみ利用する
- 作成後も編集可能（種別とは異なり変更制限なし）
- 使用可能な値はコード（`CategoryIcons` / `CategoryColors`）を参照
